# PostHog post-wizard report

The wizard has completed a deep integration of PostHog analytics into the Prepzo frontend. PostHog is now initialized with error boundary tracking in `main.jsx`, and 7 custom events are captured across 3 key pages covering the full user journey from plan creation through chat engagement.

## Changes made

| File | Change |
|------|--------|
| `src/main.jsx` | Added `posthog.init`, `PostHogProvider`, and `PostHogErrorBoundary` wrapping the app |
| `src/pages/InputPage.jsx` | Added `plan_generated`, `pdf_uploaded`, and `pdf_analysis_viewed` event captures |
| `src/pages/ResultPage.jsx` | Added `plan_viewed`, `chat_opened`, and `filter_applied` event captures |
| `src/pages/ChatPage.jsx` | Added `chat_message_sent` event capture |
| `.env` | Added `VITE_PUBLIC_POSTHOG_TOKEN` and `VITE_PUBLIC_POSTHOG_HOST` |

## Events tracked

| Event | Description | File |
|-------|-------------|------|
| `plan_generated` | User successfully generates a study plan. Key conversion event. | `src/pages/InputPage.jsx` |
| `pdf_uploaded` | User successfully uploads and processes a PDF syllabus. | `src/pages/InputPage.jsx` |
| `pdf_analysis_viewed` | User clicks 'View Full Analysis' to see ML clustering and pattern results. | `src/pages/InputPage.jsx` |
| `plan_viewed` | User lands on the ResultPage to view their generated plan. Top of post-generation funnel. | `src/pages/ResultPage.jsx` |
| `chat_opened` | User clicks 'Open Chat' from the ResultPage to chat with Prepzo. | `src/pages/ResultPage.jsx` |
| `filter_applied` | User applies a filter on the ResultPage to narrow down questions. | `src/pages/ResultPage.jsx` |
| `chat_message_sent` | User sends a message to the Prepzo chat bot. | `src/pages/ChatPage.jsx` |
